### When to Update

- **Bug fixes** → Update TROUBLESHOOTING.md

- **Architecture changes** → Update docs/architecture.md

### Documentation Checklist

Before merging code:
- [ ] Update relevant service README

## 💡 Documentation Tips

### For Readers

1. **Start with the quick start** - Don't try to read everything
2. **Use the index** - Find what you need quickly
3. **Check troubleshooting first** - Save time on common issues
4. **Read service docs for your role** - Focus on relevant information
5. **Keep docs open** - Reference while coding

### For Writers

1. **Follow existing structure** - Maintain consistency
2. **Include code examples** - Show, don't just tell
3. **Add troubleshooting sections** - Anticipate problems
4. **Link to related docs** - Help navigation
5. **Update the index** - When adding new docs
